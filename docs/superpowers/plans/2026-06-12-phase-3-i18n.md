# Fase 3 — Internacionalização (EN / ES / DE) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]` checkboxes.

**Goal:** Tornar o site **completo em PT, EN, ES e DE** — roteamento por idioma (PT na raiz, demais prefixados), **slugs localizados**, **seletor de idioma no header**, `hreflang`/sitemap por idioma, dicionários traduzidos (marcados p/ revisão), e **blocos regionais híbridos** por mercado.

**Architecture:** Astro i18n (`defaultLocale: 'pt'`, `prefixDefaultLocale: false`). Cada página vira um **template `locale`-aware** (corpo extraído, usa `t(locale, key)`); rotas geradas por **wrappers finos** por idioma com **slug no nome do arquivo** (PT na raiz + `en/`, `es/`, `de/`). Um **mapa de rotas único** (`src/i18n/routes.ts`) liga pageKey → slug por idioma, usado pelos wrappers, pelo seletor e pelo hreflang. Conteúdo continua estático (SSG).

**Tech Stack:** Astro 5 i18n, os tokens/temas/Roboto da Fase 2.5, `@astrojs/sitemap` (por locale), astro-icon.

**Decisões (do briefing):** paridade total no lançamento; reaproveitar copy EN/ES do site atual (`/remote-support-usa/`, `/remote-support-latam/`) + blog EN; **DE** traduzido e marcado p/ **revisão profissional**; **posicionamento híbrido** (base traduzida + blocos regionais: EN→EUA/Ocoee-FL, ES→LATAM, DE→DACH, PT→Brasil).

## Arquitetura i18n

**`astro.config.mjs`:**
```js
i18n: {
  defaultLocale: 'pt',
  locales: ['pt', 'en', 'es', 'de'],
  routing: { prefixDefaultLocale: false, redirectToDefaultLocale: false },
}
```

**`src/i18n/routes.ts`** — mapa único pageKey → slug por idioma (sem o prefixo de locale):
```ts
export const ROUTES = {
  home:      { pt: '',                     en: '',                  es: '',                   de: '' },
  servicos:  { pt: 'servicos',             en: 'services',          es: 'servicios',          de: 'leistungen' },
  pecas:     { pt: 'pecas-de-reposicao',   en: 'spare-parts',       es: 'piezas-de-repuesto', de: 'ersatzteile' },
  sistemas:  { pt: 'sistemas-roboticos',   en: 'robotic-systems',   es: 'sistemas-roboticos', de: 'robotersysteme' },
  pushcorp:  { pt: 'solucoes-pushcorp',    en: 'pushcorp',          es: 'pushcorp',           de: 'pushcorp' },
  sobre:     { pt: 'sobre-nos',            en: 'about-us',          es: 'sobre-nosotros',     de: 'ueber-uns' },
  carreiras: { pt: 'trabalhe-conosco',     en: 'careers',           es: 'trabaja-con-nosotros', de: 'karriere' },
  contato:   { pt: 'contato',              en: 'contact',           es: 'contacto',           de: 'kontakt' },
  privacidade:{ pt:'politica-de-privacidade', en:'privacy-policy',  es:'politica-de-privacidad', de:'datenschutz' },
  cookies:   { pt: 'politica-de-cookies',  en: 'cookie-policy',     es: 'politica-de-cookies', de: 'cookie-richtlinie' },
} as const;
// helper: localizedPath(pageKey, locale) → '/en/services/' (pt → '/servicos/'); home pt → '/'
```

**Templates + wrappers (DRY):**
- Extrair o corpo de cada página atual (`src/pages/X.astro`) para um template `locale`-aware em **`src/page-templates/<Page>.astro`** (prop `locale`, usa `t(locale, ...)` e os blocos regionais).
- **Rotas PT (raiz):** `src/pages/servicos.astro` etc. viram wrappers finos: `---import S from '@/page-templates/Servicos.astro'---<S locale="pt" />`.
- **Rotas EN/ES/DE:** arquivos com o **slug localizado** no nome: `src/pages/en/services.astro` → `<S locale="en" />`, `src/pages/es/servicios.astro`, `src/pages/de/leistungen.astro`. (11 páginas × 3 = 33 wrappers + 11 PT.)
- `trailingSlash: 'always'` mantido.

**`t()` e dicionários:** estender `src/i18n/index.ts` para registrar `{pt,en,es,de}`; `src/i18n/{en,es,de}.ts` espelham a estrutura de `pt.ts`.

**LanguageSwitcher** (`src/components/LanguageSwitcher.astro`, no header): ícone globo (`solar:global-linear`) + menu PT/EN/ES/DE; usa a pageKey atual + `ROUTES` p/ linkar a **mesma página** no idioma destino; marca o idioma ativo; a11y (combobox/menu, teclado). A pageKey atual é passada via prop pelo template.

**hreflang & sitemap:** componente `Hreflang.astro` no `<head>` (via BaseLayout) gera `<link rel="alternate" hreflang="x">` para os 4 idiomas + `x-default` (PT), usando `ROUTES`. `@astrojs/sitemap` com `i18n` configurado.

**`<html lang>`** por locale (já há o map no BaseLayout). Strings de UI e copy de página vêm de `t(locale, ...)`.

## Abordagem de tradução
- **Fonte:** `pt.ts` é a base. Reaproveitar a copy real **EN** (`/remote-support-usa/`) e **ES** (`/remote-support-latam/`) do site atual onde encaixa (fetch). **DE** é novo.
- Cada idioma: um subagente traduz `pt.ts` → `{en,es,de}.ts` mantendo as chaves, tom técnico, e **marcando tudo `// REVISAR` (DE: `// REVISÃO PROFISSIONAL`)**. Não traduzir nomes próprios/marcas/termos KUKA.
- **Blocos regionais** (config por locale): contato/endereço (PT→Joinville; EN→Ocoee/FL; ES→contato LATAM; DE→DACH *(a confirmar)*), indústrias-foco, e sub-headline opcional.
