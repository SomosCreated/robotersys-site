# Fase 2.5 — Marca & UX (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]` checkboxes.

**Goal:** Aplicar a marca e UX que faltavam: **fonte Roboto** (oficial), **light/dark mode** (dark padrão + toggle), **botão WhatsApp flutuante** chamativo, e trocar placeholders pelas **imagens reais do site atual**. (O **seletor de idioma** entra na Fase 3.)

**Architecture:** Cores viram **tokens semânticos** via CSS variables que trocam por tema (`.dark` padrão / `.light`), mapeados como cores do Tailwind. Um script inline no `<head>` aplica o tema **antes do paint** (sem flash). Refatoramos componentes/páginas para usar os tokens em vez de cores hardcoded. Tudo continua estático.

**Tech Stack:** Astro 5, Tailwind v3 (`darkMode: 'class'` não é usado — usamos tokens via CSS vars), `@fontsource-variable/roboto` + `@fontsource-variable/roboto-mono` (fallback estático se a variável não existir), astro-icon.

**Decisões (do briefing):** dark é o padrão/identidade; toggle no header; respeita `prefers-color-scheme` na 1ª visita; escolha salva em `localStorage`. Roboto (corpo/títulos) + Roboto Mono (rótulos técnicos). WhatsApp flutuante alterna sirene → "LINHA PARADA" → ícone WhatsApp → "ENTRE EM CONTATO", com `prefers-reduced-motion` calmando. Imagens do `robotersys.com`.

## Sistema de tokens (o coração da fase)

Em `src/styles/global.css`, definir variáveis para os dois temas (canais RGB para suportar `/<alpha>`):

```css
:root, .dark {            /* DARK — padrão/identidade */
  --color-bg: 5 5 5;            /* #050505 */
  --color-surface: 8 8 8;      /* #080808 */
  --color-elevated: 24 24 27;  /* zinc-900 */
  --color-line: 39 39 42;      /* zinc-800 */
  --color-strong: 255 255 255; /* títulos/foreground */
  --color-base: 212 212 216;   /* corpo (zinc-300) */
  --color-muted: 161 161 170;  /* secundário (zinc-400) */
  --color-accent: 234 88 12;   /* orange-600 */
  --color-ok: 34 197 94;       /* green-500 */
}
.light {                  /* LIGHT */
  --color-bg: 250 250 250;     /* #fafafa */
  --color-surface: 244 244 245;/* zinc-100 */
  --color-elevated: 228 228 231;/* zinc-200 */
  --color-line: 212 212 216;   /* zinc-300 */
  --color-strong: 24 24 27;    /* zinc-900 */
  --color-base: 39 39 42;      /* zinc-800 */
  --color-muted: 82 82 91;     /* zinc-600 (≥4.5:1 no claro) */
  --color-accent: 234 88 12;   /* mantém laranja */
  --color-ok: 22 163 74;       /* green-600 (contraste no claro) */
}
```

Em `tailwind.config.mjs`, mapear:
```js
colors: {
  bg: 'rgb(var(--color-bg) / <alpha-value>)',
  surface: 'rgb(var(--color-surface) / <alpha-value>)',
  elevated: 'rgb(var(--color-elevated) / <alpha-value>)',
  line: 'rgb(var(--color-line) / <alpha-value>)',
  strong: 'rgb(var(--color-strong) / <alpha-value>)',
  base: 'rgb(var(--color-base) / <alpha-value>)',
  muted: 'rgb(var(--color-muted) / <alpha-value>)',
  accent: 'rgb(var(--color-accent) / <alpha-value>)',
  ok: 'rgb(var(--color-ok) / <alpha-value>)',
}
```

**Mapa de refatoração (old → token):**
| Hardcoded (dark) | Token |
|---|---|
| `bg-[#050505]`, `bg-ink` | `bg-bg` |
| `bg-[#080808]`, `bg-surface` | `bg-surface` |
| `bg-zinc-900`, `bg-zinc-900/30` etc. | `bg-elevated` (manter a opacidade: `bg-elevated/30`) |
| `border-zinc-800` (e `/50`) | `border-line` (`border-line/50`) |
| `text-white` **em título/texto** | `text-strong` |
| `text-white` **sobre `bg-accent`/`bg-orange` (botões)** | **mantém `text-white`** (branco nos 2 temas) |
| `text-zinc-300` | `text-base` |
| `text-zinc-400` | `text-muted` |
| `text-accent` / `bg-accent` / `orange-600` | `text-accent` / `bg-accent` (já é token) |
| `text-green-500` (status) | `text-ok` |
| `#050505`/`#080808` em gradientes/sombras inline | `rgb(var(--color-bg))` / `rgb(var(--color-surface))` |
| `.bg-grid` cor da linha | usar `var(--color-line)` |

**Caveats (não automatizar cegamente):**
- `text-white` em botões laranja **fica branco**. Só vira `text-strong` quando é título/foreground sobre o fundo.
- Glows/sombras laranja (`rgba(234,88,12,...)`) funcionam nos dois temas — manter.
- Vídeo/poster `mix-blend-screen` é para dark; no light usar opacidade/tratamento diferente (ver M5).
- Garantir **WCAG AA também no light** (o `--color-muted` no claro = zinc-600 ≈ 5.7:1).
